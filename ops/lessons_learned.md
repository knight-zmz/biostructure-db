## 本次冷启动发现

### 1. 环境假设与实际的差距

**问题**: 历史文档 (DEPLOYMENT_STATUS.md 等) 显示 PostgreSQL、Redis、Nginx 已安装运行，但实际服务器是重装后的干净 ECS，所有服务均未安装。

**教训**:
- 历史文档必须与实际环境保持同步
- 服务器重装后必须重新验证所有依赖
- 部署文档应包含"从零开始"的完整流程

**改进**:
- [ ] 清理过时历史文档或标记为"历史记录"
- [ ] 创建 `QUICKSTART_FRESH.md` 专门针对新服务器
- [ ] 在 CI/CD 中加入环境验证步骤

---

### 2. 配置硬编码风险

**问题**: 数据库密码 `MyApp@2026` 硬编码在 `src/app.js` 中，存在安全风险且不利于环境隔离。

**教训**:
- 敏感配置不应提交到版本控制
- 应使用环境变量或配置管理工具
- 不同环境 (开发/测试/生产) 应有不同配置

**改进**:
- [ ] 创建 `.env.example` 模板
- [ ] 更新代码使用 `process.env` 读取配置
- [ ] 将 `.env` 加入 `.gitignore` (已存在)
- [ ] 考虑使用配置管理工具 (如 dotenv)

---

### 3. 依赖管理

**问题**: 
- node_modules 已存在但无版本锁定验证
- Python 3.6.8 较老但未确认是否需要
- PM2、PostgreSQL、Redis 未安装但项目依赖

**教训**:
- 应明确区分"开发依赖"和"运行时依赖"
- 系统级依赖 (PostgreSQL, Redis) 应有安装脚本
- 应有依赖健康检查机制

**改进**:
- [ ] 创建 `scripts/install-deps.sh` 一键安装所有依赖
- [ ] 在 README 中明确系统依赖清单
- [ ] 添加依赖验证脚本 `scripts/verify-deps.sh`

## 知识沉淀

### PostgreSQL 安装 (Alibaba Cloud Linux 3)

```bash
# 安装
sudo dnf install -y postgresql-server postgresql-contrib

# 初始化
sudo postgresql-setup --initdb

# 启动
sudo systemctl start postgresql
sudo systemctl enable postgresql
```

### Redis 安装

```bash
# 安装
sudo dnf install -y redis

# 启动
sudo systemctl start redis
sudo systemctl enable redis
```

### PM2 安装

```bash
# 全局安装
sudo npm install -g pm2

# 配置开机启动
pm2 startup
pm2 save
```
